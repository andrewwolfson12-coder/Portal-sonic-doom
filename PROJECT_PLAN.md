# Project plan and milestones

This file outlines the plan, milestones, and immediate next steps for the Portal-Sonic-Doom prototype.

High-level goals
- Engine: Unity (URP)
- Targets: WebGL (GitHub Pages) + Android (APK)
- Assets: Only free / CC0 / permissively licensed assets
- Scope: Polished vertical slice (3–7 minutes) combining traversal (Sonic), portals (Portal), and combat (Doom)

Milestones
- Phase 0 — Project skeleton & asset collection (Done: basic repo skeleton). Duration: 1–2 days
- Phase 1 — Core movement & traversal (player controller, ramps, boosters). Deliverable: playable movement demo. Duration: 4–6 days
- Phase 2 — Portals (placeable portals, teleport preserving momentum, portal camera). Deliverable: portal demo + testbed. Duration: 5–7 days
- Phase 3 — Combat (weapon, enemy AI, health, VFX). Deliverable: combat arena. Duration: 4–6 days
- Phase 4 — Level design & polish (single level combining all). Duration: 6–8 days
- Phase 5 — Optimization & cross-build (WebGL + Android builds). Duration: 3–5 days

Timeline summary: ~4–5 weeks total to a polished vertical slice.
